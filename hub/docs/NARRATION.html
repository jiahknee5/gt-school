on screen — /dev/agents

Then the agent graph — the pipeline behind Ask-the-Hub. It validates the request; builds a de-identified snapshot for retrieval, scoped to your role; checks policy — whether that role is even allowed to answer this; classifies the agent — here, the growth-strategy agent, based on the question; works out the intent, like "what should leadership focus on in Monday's meeting"; retrieves the knowledge with citations; synthesizes the answer; composes the payload; and finally scans the output for leakage — no PII.

Below that are the eval cases and the per-node rows. The key point is that every retrieval has to carry citations — on the retrieval node you can see exactly where it pulled its data.

on screen — /dev/rubrics

Rubrics and provenance — this is the claim generator, every model call behind the board. Ask-the-Hub uses an LLM and has its own graph; the smaller claims do too. Take the north-star metric — deposits against the 180 goal. The rubric says: cite the Supabase data, and use a running sum as of the selected reporting week — never the whole-dataset total — so it stays honest and it's deterministic, the same result every run.

Same for sync parity — it must cite HubSpot's data, measured not faked, and when a field is below threshold it has to say whether that's unknown, unreliable, expected, or a surprise. And for each one, the deterministic build shows the node's input, its output, and the expected output.
