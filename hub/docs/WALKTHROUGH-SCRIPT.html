5 · DATA PAGES~1:30

/dev → /dev/data-model → /dev/dictionary → /dev/integrations

Talking points (one per tab)

overviewWhat's real and what's synthetic, by source — every row on one of two join keys, reproducible from one seed.

data modelThe schema in a sentence — family → program → enrollment → payment — each table tagged for who owns the field and how it's guarded.

dictionaryEvery column documented: type, owner, protection. Nothing undocumented.

integrationsEvery source we touch, honest about maturity — connected, stand-in, or deferred — with owners, join keys, last sync. When Supabase and HubSpot disagree, this is what raises the data-confidence banner.
